# AI 工具导航平台 - 项目总览文档

> 版本：V1.0 | 日期：2026-06-03 | 状态：规划完成，待开发

---

## 一、项目背景

### 1.1 为什么要做这个项目

为 **AI 项目经理（AI PM）** 面试打造一个完整的面试作品集项目。面试官能看到的不只是代码，而是从需求分析到项目交付的**全流程能力**。

### 1.2 项目定位

一个 **AI 工具发现与导航网站**，类似"AI 版的去哪儿网"——帮用户浏览、搜索、发现各类 AI 工具。

### 1.3 技术栈

| 层次 | 技术 | 为什么选 |
|------|------|---------|
| 前端 | Vue 3 + Tailwind CSS | 之前学过 Vue，捡起快；国内面试主流 |
| 后端 | Python FastAPI | 学过 Python，自动生成 API 文档 |
| 数据库 | SQLite | 零配置，单文件，适合个人项目 |
| 开发工具 | AI 辅助编码 | 弥补技术生疏，提高效率 |

---

## 二、项目范围

### MVP 功能（必须做）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| 🔴 P0 | 首页展示 | 热门推荐 + 分类快速入口 |
| 🔴 P0 | 分类浏览 | 按 LLM、绘画、视频等分类筛选 |
| 🔴 P0 | 搜索功能 | 实时搜索，300ms 防抖 |
| 🔴 P0 | 工具详情 | 查看工具介绍、评分、官网链接 |
| 🟡 P1 | 提交工具 | 表单提交新工具推荐 |
| 🟡 P1 | 关于页面 | 项目介绍 |

### 不做功能（Post-MVP）

| 功能 | 为什么不现在做 |
|------|---------------|
| 用户注册登录 | 增加复杂度，面试价值提升有限 |
| 评分评论系统 | MVP 用固定评分，交互够用 |
| 管理后台 | 单人项目不需要 |
| 数据分析 | 需要用户量才有意义 |

---

## 三、页面结构

```
首页        /                     → 热门工具 + 分类入口
分类页      /category/:slug       → 某分类下的工具列表
详情页      /tool/:id             → 工具完整信息
搜索页      /search?q=关键词       → 搜索结果
提交页      /submit               → 提交新工具表单
关于页      /about                → 项目介绍
404页       /*                    → 未知路由处理
```

## 四、数据模型

### 分类（Category）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| name | str | 名称（如"大语言模型"） |
| slug | str | URL 别名（如"llm"） |
| description | str | 描述 |
| icon | str | 图标 |

### 工具（Tool）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| name | str | 工具名称 |
| description | str | 功能介绍 |
| url | str | 官网链接 |
| category_id | int | 所属分类（外键） |
| image_url | str | 图片链接（picsum.photos） |
| rating | float | 评分（1-5） |
| is_featured | bool | 是否首页推荐 |
| created_at | datetime | 创建时间 |

### 种子数据（20-30 条）

| 分类 | 示例工具 |
|------|---------|
| LLM / 对话 | ChatGPT, Claude, 文心一言, 通义千问, Gemini |
| AI 绘画 | Midjourney, DALL-E 3, Stable Diffusion, Leonardo.ai |
| AI 视频 | Runway, Pika, Sora, CapCut |
| 代码助手 | GitHub Copilot, Cursor, Tabnine, Codeium |
| AI 音频 | Suno, ElevenLabs, Mubert, Adobe Podcast |
| 其他 | Notion AI, Perplexity, Gamma, Canva AI |

## 五、开发计划（5 个阶段）

```
阶段一：基础搭建（第1周）
├── 任务1.1 项目脚手架搭建（2-4h）
├── 任务1.2 数据库模型设计（1-2h）
└── 任务1.3 种子数据填充（1-2h）

阶段二：后端 API（第2-3周）
├── 任务2.1 分类 API 接口（1-2h）
├── 任务2.2 工具 API 接口（2-3h）
└── 任务2.3 提交工具 API 接口（1-2h）

阶段三：前端页面（第3-5周）
├── 任务3.1 布局与导航（2-4h）
├── 任务3.2 首页（2-4h）
├── 任务3.3 分类浏览页（1-2h）
├── 任务3.4 工具详情页（1-2h）
├── 任务3.5 搜索功能（2-3h）
├── 任务3.6 提交工具表单（1-2h）
└── 任务3.7 关于页面（0.5-1h）

阶段四：质量打磨（第6周）
├── 任务4.1 响应式适配（2-3h）
├── 任务4.2 错误处理（1-2h）
└── 任务4.3 性能优化（0.5-1h）

阶段五：面试准备（第7-8周）
├── 任务5.1 项目文档完善
├── 任务5.2 面试演示材料
└── 任务5.3 Demo 部署
```

**总计预估：约 25-35 小时（每天 1-2 小时，6-8 周）**

## 七、风险管理

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| 技术遗忘（Vue/Python 生疏） | 🔴 高 | AI 辅助编码 + 先搭最小可运行例子 |
| 范围膨胀 | 🟡 中 | 严格 MVP，新功能记到"下次做"列表 |
| 时间不足（每天 1-2h 被冲掉） | 🟡 中 | 任务控制在 30min-2h，利用零碎时间 |
| 环境搭建卡住 | 🟡 中 | 用最新稳定版，先查文档再问 AI |
| 部署失败 | 🟢 低 | 截图兜底，本地能跑最重要 |
| 面试讲不清楚 | 🟡 中 | 准备 5 分钟脚本 + Q&A，提前模拟 |

## 八、面试价值

### 面试官可能会问的问题

**产品类**
- "为什么选这个项目？" → 贴合 AI PM 岗位，展示 AI 行业理解
- "怎么确定功能优先级？" → P0/P1 分级，确保 MVP 可用
- "为什么不做注册登录？" → MVP 原则，核心价值是浏览发现

**管理类**
- "项目遇到什么困难？" → 技术遗忘靠 AI 辅助，时间有限靠任务拆解
- "怎么控制范围？" → 严格 MVP，增强功能标记为 Post-MVP
- "怎么保证质量？" → 每个任务有验收标准 + 手动测试清单

**技术类**
- "为什么选 Vue 3 / FastAPI / SQLite？" → 见"关键决策"表格

### 面试展示流程

1. **30秒开场**：介绍项目是什么
2. **1-2分钟 PM 思考**：为什么选这个方向、如何定范围、怎么拆任务
3. **1分钟管理亮点**：风险管理、决策日志、范围控制
4. **1-2分钟功能演示**：首页、搜索、分类、提交——边演示边讲
5. **30秒总结**：完整走完 0 到 1 的产品交付流程
