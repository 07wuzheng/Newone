## 面试话术准备 (5分钟演示脚本)

### 开场 (30秒)
> "我做了这个AI工具导航平台，它是一个帮助用户发现和探索AI工具的目录网站。我从需求分析开始，完成了从设计到开发到上线的全流程。"

### 作为PM的思考 (1-2分钟)
> "选这个项目是因为它贴合AI PM岗位，能展示我对AI行业的理解。在需求阶段，我定义了MVP范围，把用户故事按P0/P1/P2分级，确保核心功能优先交付。排期上考虑到每天只有1-2小时，我把任务拆成30分钟到2小时的小块，每周聚焦一个阶段。"

### 项目管理亮点 (1分钟)
> "我建立了风险管理表，预判了'技术遗忘'和'范围膨胀'两个主要风险——前者通过AI辅助编码解决，后者通过严格的MVP范围控制。项目过程中我做了决策日志，记录技术选型的原因，比如选FastAPI是因为它自动生成API文档，选SQLite是因为零配置、适合个人项目。"

### 项目成果展示 (1-2分钟)
> "这是首页，展示热门工具和分类导航。搜索功能是实时的，200ms防抖。提交工具有完整的表单验证。技术栈是Vue 3 + FastAPI + SQLite..."

### 总结 (30秒)
> "这个项目让我完整实践了一次从0到1的产品交付流程——需求分析、任务拆解、技术选型、风险管理和最终交付。我认为这些能力能直接应用到AI PM的工作中。"

## 面试常见问题 - PM角度准备

### 产品/需求类
| 问题 | 回答要点 |
|------|---------|
| 为什么选这个项目？ | 贴合AI PM岗位，能展示全流程 |
| 你怎么确定功能优先级？ | P0/P1/P2分级，确保MVP可用 |
| 用户是谁？ | 三个Persona：AI爱好者、开发者、业务用户 |
| 为什么不做注册登录？ | MVP原则，核心价值是浏览发现，认证是P2 |

### 项目管理类
| 问题 | 回答要点 |
|------|---------|
| 你遇到了什么困难？ | 技术遗忘通过AI辅助解决；时间有限通过任务拆解 |
| 如何保证进度？ | 每周一个阶段，每天1-2小时小块任务 |
| 范围怎么控制的？ | 严格的MVP范围，所有增强功能标记为Post-MVP |
| 怎么保证质量？ | 每个任务有验收标准，手动测试清单 |

### 技术类 (PM也需要懂)
| 问题 | 回答要点 |
|------|---------|
| 为什么选Vue 3？ | 组件化、Composition API更灵活、文档完善 |
| 为什么选FastAPI？ | 自动生成API文档、Python生态、性能好 |
| 为什么用SQLite？ | 零配置、文件数据库、适合个人项目 |
| 如何部署？ | 前端Vercel，后端Railway/PythonAnywhere |
