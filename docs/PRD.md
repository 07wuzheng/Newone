# AI 工具导航平台 - 产品需求文档 (PRD)

## 1. 项目背景

### 1.1 项目目标
打造一个 AI 项目经理面试作品集项目，展示从需求分析到交付的全流程能力。

### 1.2 目标用户
- **主要用户**：AI 从业者和爱好者，需要发现新 AI 工具
- **次要用户**：想对比不同 AI 工具优缺点的普通用户

## 2. 功能范围

### MVP 范围（必须做）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 首页展示 | 热门工具 + 分类入口 |
| P0 | 分类浏览 | 按分类筛选工具列表 |
| P0 | 搜索功能 | 关键词实时搜索 |
| P0 | 工具详情 | 查看工具完整信息 |
| P1 | 提交工具 | 用户推荐新工具 |
| P1 | 关于页面 | 项目介绍 |

### Post-MVP（本次不做）

| 功能 | 为什么不现在做 |
|------|---------------|
| 用户注册登录 | 增加了复杂度但对面试价值提升有限 |
| 评分/评论系统 | MVP 使用固定评分，交互足够 |
| 管理后台 | 单人项目不需要 |
| 数据统计 | 需要用户量才有意义 |
| API 限流 | 个人项目无此需求 |

## 3. 用户故事

### P0 - 核心功能
- 作为访客，我想在首页看到热门 AI 工具，以便快速了解当前有哪些好用的工具
- 作为访客，我想按分类浏览工具，以便找到我感兴趣的领域的工具
- 作为访客，我想通过搜索找到特定工具，以便快速定位我需要的信息
- 作为访客，我想查看工具详情，以便了解它的功能介绍和访问方式

### P1 - 增强功能
- 作为访客，我想推荐一个新的 AI 工具，以便和其他人分享好工具
- 作为访客，我想了解这个项目的背景，以便决定是否信任和使用

## 4. 页面结构

```
/                   → 首页（热门工具 + 分类入口）
/category/:slug     → 分类页面（该分类下的工具列表）
/tool/:id           → 工具详情页
/search?q=keyword   → 搜索结果页
/submit             → 提交工具页
/about              → 关于页面
/*                  → 404 页面
```

## 5. 非功能性需求

- **响应式**：支持手机（320px）到桌面（1440px）的所有分辨率
- **加载速度**：首页加载 < 3 秒
- **可用性**：所有表单和按钮在 1 次点击内响应
- **错误处理**：API 失败时有用户友好的错误提示

## 7. 成功标准

1. ✅ 所有 P0 功能可正常运行
2. ✅ 页面在手机和桌面端均显示正常
3. ✅ 搜索能在 1 秒内返回结果
4. ✅ 提交工具表单校验正常
5. ✅ 项目文档完整，能在面试中展示
